# Comments must explain why, not what
Reasoning: The code already describes what it does. A comment that restates the code is noise that drifts out of sync and misleads readers. The only valuable comment explains non-obvious intent.

```bash
jumbo invariant add --title "Comments must explain why, not what" --description "Do not write comments that restate what the code does. Prefer code so clear that no explanation is needed. The only acceptable comment is one that explains the reasoning behind a non-obvious decision — why a workaround exists, why a constraint is in place, or why the obvious alternative was rejected. Commented-out code must be deleted."
```
